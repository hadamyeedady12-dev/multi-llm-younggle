# Multi-LLM Council Skill

다중 LLM CLI 도구를 활용한 집단 지성 시스템입니다.

## 개요

Claude Code가 의장으로서 Gemini CLI, Codex CLI를 조율하여 3단계 협의 프로세스를 진행합니다. 추가 API 비용 없이 기존 CLI 구독만으로 운영됩니다.

## 사용 가능한 명령어

| 명령어 | 설명 |
|--------|------|
| `/council <질문>` | 표준 3단계 협의 |
| `/council-quick <질문>` | 신속 협의 (Stage 2 생략) |
| `/council-expert <질문>` | 전문가 패널 (역할 기반) |

## 위원 구성

| 위원 | CLI | 모델 |
|------|-----|------|
| Gemini | `gemini --yolo` | gemini-3-pro-preview |
| Codex | `codex exec` | gpt-5.2-codex |
| Claude | (의장) | claude-opus-4.5 |

## 3단계 프로세스

```
┌─────────────────────────────────────────────┐
│           Multi-LLM Council                  │
├─────────────────────────────────────────────┤
│                                             │
│  Stage 1: 독립적 의견 수집                    │
│  ├── Gemini CLI 호출                         │
│  ├── Codex CLI 호출                          │
│  └── Claude 자체 의견 생성                    │
│                                             │
│  Stage 2: 익명 교차 평가                      │
│  ├── 응답 익명화 (A, B, C)                   │
│  ├── 각 위원이 타 응답 평가                   │
│  └── 순위 집계                               │
│                                             │
│  Stage 3: 최종 종합                          │
│  ├── 합의점 도출                             │
│  ├── 이견 명시                               │
│  └── 신뢰도 평가                             │
│                                             │
└─────────────────────────────────────────────┘
```

## 핵심 원칙

1. **독립성**: 각 위원은 독립적으로 의견 제시
2. **익명성**: 평가 시 응답자 신원 비공개
3. **투명성**: 모든 과정과 근거 공개
4. **합의 지향**: 다수결보다 논거의 질 중시

## 에러 처리

- CLI 실패 시 2회 재시도
- 2명 미만 응답 시 세션 중단
- 타임아웃: 120초/위원

## 요구사항

- Gemini CLI (`gemini`) 설치 및 인증
- Codex CLI (`codex`) 설치 및 인증
- Claude Code 환경
