# Multi-LLM Council Skill

다중 LLM CLI 도구를 활용한 집단 지성 시스템입니다.

## 개요

Claude Code가 의장으로서 Gemini CLI, Codex CLI를 조율하여 3단계 협의 프로세스를 진행합니다. 추가 API 비용 없이 기존 CLI 구독만으로 운영됩니다.

## 사용 가능한 명령어

| 명령어 | 설명 |
|--------|------|
| `/council <질문>` | 표준 3단계 협의 |
| `/council-quick <질문>` | 신속 협의 (Stage 2 생략) |
| `/council-expert <질문>` | 전문가 패널 (역할 기반) |

## 위원 구성 및 역할

| 위원 | CLI | 모델 | 핵심 역할 |
|------|-----|------|----------|
| Gemini | `gemini --yolo` | gemini-3-pro-preview | 탐색/수집/시각화 |
| Codex | `codex exec` | gpt-5.2-codex | 구현/디버깅/테스트 |
| Claude | (의장) | claude-opus-4.5 | 설계/검증/문서화 |

### Gemini - 탐색 및 시각화 전문가

- **컨텍스트 로딩**: 100만+ 토큰 처리, 전체 코드베이스 분석
- **멀티모달**: 이미지/영상 분석 → 코드 변환
- **UX/UI 디자인**: 시안, 레이아웃, 최신 트렌드 (Glassmorphism, Bento Grid)
- **실시간 검색**: Google Search 연동으로 최신 정보 확인

### Codex - 구현 및 디버깅 전문가

- **코드 구현**: 실행 가능한 코드 작성, 리팩터링
- **디버깅**: 에러 추적, 루트 원인 분석, 패치 제안
- **테스트**: 테스트 케이스 작성, 빌드/배포 자동화
- **파일 시스템**: 실제 파일/디렉토리 단위 작업 맥락 이해

### Claude - 설계 및 검증 전문가 (의장)

- **아키텍처 설계**: 기술 명세, 시스템 구조 설계
- **논리 검증**: 일관성 검토, 리스크 분석
- **문서화**: UX 카피, 사용자 흐름, 브랜드 톤
- **안전성**: 보안 이슈 검토, 윤리적 판단

## 3단계 프로세스

```
Stage 1: 독립적 의견 수집
  - Gemini CLI 호출
  - Codex CLI 호출
  - Claude 자체 의견 생성

Stage 2: 명시적 교차 평가
  - 위원 이름 명시하여 평가
  - 각 위원이 다른 두 위원 평가
  - 순위 집계 및 평가 근거 공개

Stage 3: 최종 종합
  - 합의점 도출
  - 이견 명시
  - 신뢰도 평가
```

## 협업 파이프라인

```
┌─────────────────────────────────────────────────────────────┐
│                    LLM Council Workflow                      │
├─────────────────────────────────────────────────────────────┤
│  Stage 1: CONTEXT (Gemini)                                  │
│  - 컨텍스트 로딩, 멀티모달 분석, UX/UI 시안                   │
├─────────────────────────────────────────────────────────────┤
│  Stage 2: DESIGN (Claude)                                   │
│  - 아키텍처 설계, 논리 검증, 명세 작성                        │
├─────────────────────────────────────────────────────────────┤
│  Stage 3: IMPLEMENT (Codex)                                 │
│  - 코드 구현, 디버깅, 테스트                                  │
├─────────────────────────────────────────────────────────────┤
│  Cross-Validation Loop                                      │
│  Codex 결과 → Claude 리뷰 → Gemini UX 검증 → 완료           │
└─────────────────────────────────────────────────────────────┘
```

## 디버깅 협업 플로우

```
버그 발생
    ↓
[Codex] 에러 로그 분석 → 루트 원인 파악 → 패치 제안
    ↓
[Claude] 수정안 논리 검토 → 부작용/보안 이슈 확인
    ↓
[Gemini] 최신 라이브러리 확인 → Best Practice 검색
    ↓
최종 수정 적용
```

## 핵심 원칙

1. **독립성**: 각 위원은 독립적으로 의견 제시
2. **명시성**: 평가 시 위원 이름을 명확히 표시
3. **투명성**: 모든 과정과 근거 공개
4. **합의 지향**: 다수결보다 논거의 질 중시

## 에러 처리

- CLI 실패 시 2회 재시도
- 2명 미만 응답 시 세션 중단
- 타임아웃: 120초/위원

## 요구사항

- Gemini CLI (`gemini`) 설치 및 인증
- Codex CLI (`codex`) 설치 및 인증
- Claude Code 환경
