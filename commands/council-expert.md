# /council-expert - 전문가 패널

각 위원에게 특정 전문가 역할을 부여하여 다각적 분석을 수행합니다.

## 사용법
```
/council-expert <질문>
```

## 실행 지침

$ARGUMENTS 를 사용자 질문으로 처리하세요.

---

## 역할 기반 협의 프로세스

### 역할 부여

| 위원 | 역할 | 관점 |
|------|------|------|
| Gemini | 보안/성능 전문가 | 시스템 안정성, 보안 취약점, 성능 최적화 |
| Codex | 구현/실용성 전문가 | 코드 품질, 구현 복잡도, 유지보수성 |
| Claude | UX/아키텍처 전문가 | 사용자 경험, 시스템 설계, 확장성 |

### Stage 1: 역할 기반 의견 수집

**1. Gemini (보안/성능 전문가):**
```bash
gemini "당신은 보안 및 성능 전문가입니다. 다음 질문을 보안 취약점, 성능 병목, 시스템 안정성 관점에서 분석해 주세요.

질문: {사용자질문}

분석 형식:
1. 보안 관점 분석
2. 성능 관점 분석
3. 권장사항" --yolo
```

**2. Codex (구현/실용성 전문가):**
```bash
codex exec "당신은 구현 및 실용성 전문가입니다. 다음 질문을 코드 품질, 구현 복잡도, 유지보수성 관점에서 분석해 주세요.

질문: {사용자질문}

분석 형식:
1. 구현 복잡도 분석
2. 코드 품질 고려사항
3. 권장사항"
```

**3. Claude (UX/아키텍처 전문가):**
사용자 경험, 시스템 설계, 확장성 관점에서 분석합니다.

### Stage 3: 다각적 종합

의장이 모든 전문가 의견을 종합하여 균형 잡힌 결론 도출.
